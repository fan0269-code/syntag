# Prompt 08/08：发布前全量验收与交接

## 目标

只在前 01–07 步完成后执行。以证据决定是否可称为“完整网站”，而不是只凭页面看起来正常或构建通过。

## 允许范围

本步骤默认只做验证、修复本验收发现的明确回归、记录结果。不要新增功能、重构架构或扩充内容范围。若发现超出前序范围的新需求，列入后续清单并停止扩张。

## 自动化验收

依次运行并记录完整退出码：

```bash
npm test
npm run lint
npm run build
```

在已配置的非生产数据库环境，再执行：

```bash
npm run db:seed
npm run db:seed
```

并用只读查询/健康脚本核对：已发布学科数、12 个主理论、学者/著作/主题/概念关系、L1 来源数、三种图谱模式的节点/边非空。若没有 `DATABASE_URL`，验收必须结论为“不可发布，等待数据库”，不得以静态构建代替。

## 浏览器验收矩阵

在本地生产构建或等价预览中检查：

1. 首页默认图谱：显示真实数据，点击节点打开详情，跳转正文。
2. 每个公开学科 × genealogy/scholars/topics：有数据即显示图谱，无数据有明确非错误状态。
3. 首页导航、页脚、六类实体索引、至少各一篇详情页、About、Privacy、Terms、Editorial Policy、404、服务不可用页。
4. 搜索：已知理论、已知学者、无结果、服务不可用。
5. 站点地图、robots、OG 图片端点与一个 JSON-LD 页面。
6. 375px 与桌面宽度：导航、图谱控制、详情抽屉、搜索浮层、表格没有横向滚动和不可点击元素。

## 发布判定

只有下列条件全满足，才可写“可发布”：

- 测试、lint、build 全绿；
- 真实数据库已迁移、seed 两次成功；
- 首页不依赖未标注 demo 数据；
- 公开链接无死链；
- 真实内容页、搜索、三种图谱模式可用；
- 内容来源和 L1/L2/L3 标注通过审查；
- sitemap 只列可访问已发布页；
- README、环境变量与法律信息足以支持当前发布方式。

任何一项失败时，输出“暂不可发布”，并按 P0/P1/P2 列出：现象、复现路径、影响、建议负责步骤。不要为了关闭验收而弱化门槛或删除测试。

## 最终交接文档

在 `docs/` 新建或更新一份发布验收记录，包含：日期、代码版本/分支、环境类别（不含密钥）、执行命令与结果、浏览器矩阵、数据数量、已知限制、发布判定、下一步。未经用户明确授权，不执行部署、提交或推送。
