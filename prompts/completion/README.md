## 通用执行规则

1. 先读本文件、项目根目录 `AGENTS.md`、`package.json`、相关实现和测试；若旧提示词与现有代码冲突，以当前代码和本组提示词为准。
2. 不擅自改动未提交的无关文件；工作树当前可能已有他人修改。只修改本步骤明确列出的边界内文件。
3. 不凭空生成数据库地址、Supabase 密钥、DOI、出版信息、作者履历、来源 URL、发布日期或研究结论。需要外部事实时，使用权威/原始来源并保留来源。
4. 任何需要 `DATABASE_URL` 的步骤：缺少连接串时，只完成静态检查和准备工作，明确报告“等待配置”，不得声称已经迁移或灌入数据。
5. 每一步结束必须报告：修改文件、未修改的原因、执行命令及结果、剩余阻塞项。不要顺带实现后续步骤。
6. 未经用户明确要求，不提交、推送、部署、上传数据或创建外部资源。
