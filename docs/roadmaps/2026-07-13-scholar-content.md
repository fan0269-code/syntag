# C6 学者内容：实施计划

> 日期：2026-07-13  
> 状态：已完成（本地，未部署）  
> 阶段：C6  
> 关联：用户提供的 `Prompt C6`；`docs/roadmaps/2026-07-13-works-concepts-layer.md`

## 1. 目标与范围

- 先深化已有的 Glen H. Elder Jr.、Geert Kelchtermans、Anthony Giddens、Pierre Bourdieu 四个学者页，使其成为有来源、可追溯的学术贡献页。
- 审核 12 个理论现有核心来源中的作者与贡献者；仅当身份、理论贡献和代表作均有独立权威记录时，才考虑新增档案。

## 2. 内容合同

- 学术定位与研究领域，以及与本站理论的具体、有限关系。
- 代表著作及贡献、已核验的影响/发展/批评或合作关系，以及明确的归因边界。
- L1 仅用于书目、身份、作者归属和来源明确支持的关系；L2 为编辑性解释；L3 为研究者使用建议或待具体研究核验的判断。

## 3. 执行顺序与验收

1. 建立基于原著、DOI、出版社、大学、期刊或权威机构的四位现有学者证据包，并逐项审核其他作者的发布资格。
2. 先写失败回归：四页内容合同、理论/著作关系、归因边界、L1/L2/L3 和无占位渲染。
3. 在 `src/data/seed-content.ts` 完成作者语料；由 seed 适配数据库，并接入详情页。
4. 仅发布经过资格审核的新增学者；未满足条件的作者明确保留为不发布。
5. 运行内容验证、全量测试、lint、seed、build、差异检查与四个代表学者页浏览器检查。

## 4. 边界

- 不根据常识补充出生信息、任职经历、合作关系或未被来源明确支持的理论创始人身份。
- 不把“被本站引用”或“著作作者”自动写成理论创始人。
- 不进入 C7，也不提交、推送、部署或写入远程数据库。

## 5. 完成记录（验收后填写）

- 实际改动：在 `docs/research/2026-07-13-c6-scholars.md` 建立四位既有学者的 L1 证据包和 12 理论作者审计；扩展学者内容合同、seed 适配、来源呈现和详情页区块。每页现含学术定位、本站理论关系、代表著作与贡献、可证实的影响/发展/合作说明、归因边界、阅读路径及 L1/L2/L3 记录。
- 验证结果：C6 内容合同先失败后转绿；`npm run lint`、`npm run db:seed`、`npm test`（60/60）、`npm run build`、`git diff --check` 通过。浏览器逐页检查 Elder、Kelchtermans、Giddens、Bourdieu 四页，均显示 C6 区块、理论/著作入口和来源链接，且无 `being prepared` 或 `Verification pending`。
- 新增或不新增学者的审核结论：仅深化四个已有档案；12 理论中其他作者与贡献者均已审计但本批不新增，因为未为每人同时保留独立的权威身份/定位、代表作和有界关系证据包。机构作者 UNESCO、OECD 不是 Scholar 候选。
