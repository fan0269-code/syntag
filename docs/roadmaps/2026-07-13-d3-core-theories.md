# C2 两个 D3 核心理论深化：实施计划

> 日期：2026-07-13
> 状态：已完成（本地验收通过）
> 阶段：C2
> 负责人：Codex
> 关联：用户提供的 `Prompt C2`；`docs/roadmaps/2026-07-13-content-presentation-baseline.md`

## 1. 目标与用户价值

- 将 Life Course Theory 与 Teacher Identity Theory 建成可用于理论选择、研究设计转换和来源追溯的 D3 标杆页面。
- 让研究者不仅理解概念，还能判断理论适用边界、相邻理论差异、可观察维度和章节使用方式。

## 2. 范围

### 包含

- 两篇 D3 理论的英文语料深化、权威来源研究记录、L1/L2/L3 核验与相关测试。
- D3 专属的核心问题、发展阶段、关键学者与代表作、相邻理论比较、批评与边界区块。
- 两篇页面的逐页内容与浏览器审查。

### 不包含

- 其余 10 个理论、Psychology、Management、Works/Concepts 独立实体建设或其他产品功能。
- 视觉重设计、数据库结构或远程数据写入、提交、推送或部署。

## 3. 当前证据与决策

- C1 已证明两篇 D3 页面可以正确显示现有结构化语料，但当前每篇只有 2 个概念、1 个来源、1 条阅读路径和模板化研究建议，尚未达到 C2 深度合同。
- `src/data/seed-content.ts` 继续作为唯一作者语料源；`prisma/seed.ts` 不承载正文。
- 精确事实只使用原著、DOI、出版社、大学、期刊或权威机构记录；编辑比较标为 L2，研究设计建议标为 L3。
- Teacher Identity Theory 按多元研究传统处理，不包装成单一统一理论或把单一学者写成唯一创始人。

## 4. 实施步骤

1. 建立两份权威来源证据包 — 文件：`docs/research/2026-07-13-*-c2.md` — 产物：逐项事实与来源映射。
2. 增加 D3 深度合同与失败测试 — 文件：`src/data/templates/theory-template.ts`、`tests/content-validation.test.ts` — 产物：12 项要求可自动验收。
3. 深化两篇语料 — 文件：`src/data/seed-content.ts` — 产物：多来源、非模板化的 D3 内容。
4. 接入 D3 专属区块 — 文件：`src/lib/theory-presentation.ts`、`src/components/content/TheoryArticle.tsx` — 产物：页面可浏览的深度差异。
5. 执行内容、自动化与浏览器验收 — 产物：两页逐项审查记录与 C2 是否通过的结论。

## 5. 数据、内容与安全

- 数据来源与核验等级：L1 仅用于来源直接支持的稳定事实；L2 用于本站的理论解释、比较和适用性判断；L3 用于需按研究情境和导师意见调整的数据与写作建议。
- 数据迁移/seed 影响：不改 schema；仅允许将语料同步到本机开发数据库，用于浏览器验收，不触碰远程数据库。
- 环境变量/权限需求：本地页面验收只读使用现有开发环境，不输出连接信息。
- 隐私、版权或法律影响：只写原创摘要，不复制长段原文；访问路径仅使用合法来源页面。

## 6. 验收标准

- 内容：每篇 4–6 个核心概念，至少 4 个相互独立权威来源，至少 3 级阅读路径，并覆盖 Prompt C2 的 12 项合同。
- 核验：L1 来源引用全部能回指页面来源；L2/L3 不冒充普遍有效事实。
- 自动化：定向内容测试、`npm test`、`npm run lint`、`npm run build`、`git diff --check` 全部通过。
- 浏览器：逐页检查 Life Course Theory 与 Teacher Identity Theory 的所有 D3 区块、来源、核验标签与内部链接。

## 7. 风险、回退与发布判定

- P0/P1 风险：来源不支持精确断言；Teacher Identity 被写成统一理论；研究建议被误写成普遍方法；两页复制同一套操作化文案。
- 回退方式：只回退本路线图、两份研究记录、D3 合同、两篇语料及其呈现/测试改动。
- 发布结论条件：两页逐项审查与全部门禁通过；本结论不代表 C3–C8 完成，也不改变线上未部署状态。
